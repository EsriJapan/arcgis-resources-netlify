カスタム変数を使用した解析

本演習にかかる時間はおよそ 30 分です。

前演習では、自社の顧客や売上などの独自のデータや、既存の BA 用データを使用してカスタム変数を作成しました。

本演習では、作成したカスタム変数を元にした商圏を作成したり、商圏に対してカスタム変数を集計したりする方法を学びます。

事前準備が完了していない場合は、演習に入る前にこちらの操作を行ってください。前回の演習から引き続き演習をされている方は、「限界値商圏の作成」から開始いただけます。

統計データ コレクションの追加

カスタム変数の設定がされた統計データ コレクションを、プロジェクトに追加して利用できるようにします。

  1. 「BAProチュートリアル_カスタマイズ.aprx」の [演習1] マップを開き、[コンテンツ] ウィンドウおよびマップ上に、店舗のポイント データがあらかじめ追加されていることを確認します。

  2. [解析] タブ → [ワークフロー] グループの [ビジネス解析] ボタン → [統計データ コレクションの追加] をクリックします。

  3. [既存の統計データ コレクション] ダイアログが開いたら、以下ファイルを選択します。

    ＜事前準備で演習用データを配置した場所＞\bapro_tutorial_customize\演習用データ\演習結果\町丁字等別売上情報.sdcx

  4. 手順 2 の操作を再度実行してダイアログを開き、以下のファイルをさらに選択します。

    ＜事前準備で演習用データを配置した場所＞\bapro_tutorial_customize\演習用データ\演習結果\既存のBAデータから作成した計算変数.sdcx

[カタログ] ウィンドウの [Business Analyst] → [統計データ コレクション] フォルダー内に、「町丁字等別売上情報.sdcx」と「既存のBAデータから作成した計算変数.sdcx」が追加されます。

追加した統計データ コレクションのインデックスは最新ではないため、再度インデックスを生成します。

  1. [解析] タブ → [ジオプロセシング] グループの [ツール] をクリックし、[ジオプロセシング] ウィンドウの検索ボックスに「SDCX」と入力します。
  2. 検索結果の中の [SDCX インデックスの生成 (Generate SDCX index)] ツールをクリックして開きます。
  3. [入力 SDCX ファイル] の [参照] ボタンをクリックしてダイアログを開き、以下のファイルを選択して [OK] をクリックします。

    ＜事前準備で演習用データを配置した場所＞\bapro_tutorial_customize\演習用データ\演習結果\町丁字等別売上情報.sdcx

  4. [実行] をクリックしてツールを実行します。
  5. ツール実行が完了したら、再度手順 7~8 の操作を以下のファイルに対しても同様に行います。

    ＜事前準備で演習用データを配置した場所＞\bapro_tutorial_customize\演習用データ\演習結果\既存のBAデータから作成した計算変数.sdcx

「既存のBAデータから作成した計算変数.sdcx」に関しては、日本全国のレイヤーに対してインデックスを構築するため、時間がかかる可能性があります。

限界値商圏の作成

作成したカスタム変数を利用して、限界値商圏を作成します。本演習では、自社店舗を起点として、売上高の 1 千万分をカバーするリング商圏を作成します。

限界値商圏は、起点となる店舗から外側に向かって、指定した変数の限界値に達する地点が境界線となる商圏です。たとえば、人口 10,000 人をカバーするのに最低限必要な範囲を把握することができます。 BA Pro では、シンプルなリング商圏と、道路ネットワークを元にした到達圏の 2 種類の限界値商圏を作成できます。

  1. [ジオプロセシング] ウィンドウの [戻る] ボタンをクリックし、検索ボックスに「限界値」と入力します。

  2. 検索結果の中の [限界値リングの生成 (Generate Threshold Rings)] ツールをクリックして開きます。

  3. [限界値変数] 以外のパラメーターを以下のように設定します。

    • 入力フィーチャ：自社店舗
    • 出力フィーチャクラス：売上1千万をカバーするリング商圏
    • 入力方法：値
    • 限界値：10000000
    • 距離単位：キロメートル
    • ID フィールド：店舗ID

  4. [限界値変数] の [＋] マークをクリックし、データ ブラウザーを開きます。

  5. データ ブラウザーが開いたら [カスタム データ] ノードをクリックし、「千葉県内の町丁字等別売上情報」カテゴリを開きます。

カスタム変数は、データ ブラウザーの [カスタム データ] ノード内に格納されます。

  1. カテゴリ別の変数の一覧が表示されるので、[売上] カテゴリの「売上高」を選択し、[OK] をクリックしてデータ ブラウザーを閉じます。
  2. [実行] をクリックして、ツールを実行します。

マップに、各店舗から売上の 1 千万をカバーするリング商圏が追加されます。結果の詳細を確認します。

  1. [コンテンツ] ウィンドウ上の「売上1千万をカバーするリング商圏」を右クリック → [属性テーブル] をクリックして、属性テーブルを開きます。

属性テーブルは、[コンテンツ] ウィンドウで対象のレイヤーをクリックして選択した状態で、Ctrl + T でも開くことができます。

「実際の値」フィールドに、各商圏が実際にカバーする限界値が格納されます。「半径」フィールドには、作成されたリングの半径が格納されます。 今回の結果では、売上高の 1 千万を獲得するために必要だったエリアが一番大きいのは店舗 A 、一番小さいのは店舗 C であるということが分かります。

商圏にカスタム変数を集計する

最後に、商圏に対して前演習で作成した「20~40歳代の人口」を集計します。

  1. [ジオプロセシング] ウィンドウの [戻る] ボタンをクリックし、検索ボックスに「情報付加」と入力します。
  2. 検索結果の中の [レイヤーへの情報付加 (Enrich Layer)] ツールをクリックして開きます。
  3. [入力フィーチャ] のプルダウンから「売上1千万をカバーするリング商圏」を選択します。
  4. [出力フィーチャクラス] で、「売上1千万をカバーするリング商圏_20_40代人口付与」と入力します。
  5. [変数] で [+] ボタンをクリックしてデータ ブラウザーを開き、[カスタム データ] ノード → [既存のBAデータから作成した計算変数] をクリックして開きます。
  6. [既定] カテゴリにチェックを入れて格納されているすべての変数を選択し、[OK] をクリックしてデータ ブラウザーを閉じます。
  7. [変数] で 4 つの変数が選択されていることを確認したら、[実行] をクリックしてツールを実行します。

統計データ コレクションのインデックスが未構築、もしくは最新でない場合は、以下のような警告が表示されます。
その場合は、[カタログ] ウィンドウ → [Business Analyst] → [統計データ コレクション] 配下に保存されている統計データ コレクションを右クリック → [編集] をクリックして編集ダイアログを開き、[ソース] タブの [パフォーマンス インデックス] の工具マークをクリックしてインデックスを構築した後に、ツールを再実行してください。

ツール実行が完了すると、[コンテンツ] ウィンドウおよびマップに結果レイヤーが追加されます。集計結果を確認します。

  1. [コンテンツ] ウィンドウ上の「売上1千万をカバーするリング商圏_20_40代人口付与」を選択し、Ctrl + T で属性テーブルを開き、右にスクロールします。

「2015 人口 20~40歳代」フィールド値が、以下の 3 つのフィールドの合算値であり、カスタム変数が正しく集計されていることが分かります。

  • 2015 人口 20歳代
  • 2015 人口 30歳代
  • 2015 人口 40歳代
  1. [プロジェクト] タブ → [保存] を選択し、プロジェクトを保存します。

まとめ

この演習では、カスタム変数を使用して限界値商圏を作成したり、商圏に対してカスタム変数を集計する方法を学ぶことで、カスタム変数を解析に利用する方法を学びました。

また、以下のツールの操作を学びました。

  • [SDCX インデックスの生成] ツール
  • [限界値リングの生成] ツール
  • [レイヤーへの情報付加] ツール
